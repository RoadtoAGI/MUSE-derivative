# 末世（apocalypse）世界观参考

> 本文不复述 SKILL.md §3 的"原因 vs 后果"原则、末世 4 反例、"互斥 driver 并列"禁令。

## generative_driver 选型范畴

末世 driver 必须回答四件事：

- **灾变成因** — 什么具体事件 / 物质 / 机制把世界变成末世（病原体 / 气候 / 宇宙事件 / AI 反噬 / 核污染 / 异常规则污染 / 异种入侵）
- **传播或扩散机制** — 灾变如何从局部扩到全局：传播媒介、潜伏期、何种行为加速 / 减缓
- **可信 vs 不可信资源** — 灾后哪些物 / 空间 / 食物可信、哪些被污染——这是 Phase 5/6 场景威胁与可用资源的根
- **威胁的可识别程度** — 主角能 / 不能预判威胁；威胁是否能被规则化（如 I Am Legend 类型）还是不可预判（如 The Road）

### platform_mode vs classic_mode

| 模式 | 特征 | 适用 |
|------|------|------|
| **platform_mode** | 灾变成因前置到题眼 / 标签；driver 透明，给读者掌握感 | 起点 / 番茄 / 知乎盐言主流末世 |
| **classic_mode** | 灾变成因故意隐去；驱动来自"已无法追问原因"的废墟感 | 文学型 / 严肃末世 |

默认 platform_mode。若用户明示"名著感 / 文学化"或 Phase 0 `style_directives` 含"留白 / 隐去成因"，切 classic_mode。

## 名著锚点（few-shot）

### 《I Am Legend》— vampiris bacillus 病原型 / platform_mode

driver 拆解：

- **原因**：核战后尘暴 + 昆虫变异 → 携带 vampiris bacillus（一种圆柱形细菌），通过昆虫 / 尘暴 / 直接接触传播
- **运作机制**：细菌在血液中形成等渗溶液，激活死者所有生理功能；阳光（UV）杀菌 → 感染者白天昏迷；大蒜含烯丙基硫醚 → 触发过敏；木桩有效不是因为刺心而是因为让空气进入伤口（细菌兼性腐生：体内厌氧共生 / 体外好氧寄生）
- **局部表现**：街区夜战、紫外线灯、防风暴口罩、烧尸坑、菜园种大蒜、地下储藏室、《吸血鬼科学》研究笔记

派生 world_rules 关键条：
- 物理：弹药无效（尸体腺体分泌 glue 即刻封伤口）、断粮时细菌产孢 / 噬菌体、可药控（输血 + 抑制剂）
- 社会：旧法律强制焚尸；新感染者形成有医学控制能力的军事化社会，视主角为屠杀者

参考产物：MUSE-canon-distill `knowledge-base/novels/I Am Legend/pipeline/phase1_world.yaml`。

### 《长路》(The Road) — 成因隐去型 / classic_mode

driver 拆解：

- **原因**：仅给一句"凌晨 1:17 的闪光"，**故意不解释**——driver 的"原因"在文本层被悬置
- **运作机制**：太阳遮蔽 → 光合作用停止 → 食物链断裂 → 持续降温；灰烬遮天，呼吸是慢性侵蚀
- **局部表现**：永远灰蒙的天空、灰烬覆盖的路面、防尘口罩、推车、躲避炊烟（暴露位置）、不敢生火过夜、地窖中被截肢进食的活人

派生 world_rules 关键条：
- 物理：无动物、无鸟、无鱼；海洋变灰；森林大火零星
- 社会：无政府无货币；"好人 vs 食人族"二分；信任本身是致命风险
- 心理：梦见旧世界 = 放弃求生意志的预兆；"火炬"是父子间的善良符号

参考产物：MUSE-canon-distill `knowledge-base/novels/长路/pipeline/phase1_world.yaml`。

### 《流浪地球》— 太阳氦闪型（科幻末世杂交）

driver 拆解：

- **原因**：太阳将在 400 年内发生氦闪 → 地球必须逃离太阳系
- **运作机制**：12000 台地球发动机（重元素聚变）安装在亚洲 / 美洲板块；分五步走（刹车 → 逃逸 → 流浪Ⅰ/Ⅱ → 新太阳）；绕太阳 15 圈变轨 + 木星引力弹弓
- **局部表现**：地球自转停止 → 一面永昼一面永夜；发动机区酷热（70-80°C）；远日点大气冻结为固态氧氮；地下城

派生 world_rules 关键条（部分）：
- 物理：自转停止 / 大气冻结 / 木星借力的物理约束
- 社会：联合政府统治；危急法（撤离按年龄，幼者优先）；生育权抽签

参考产物：MUSE-canon-distill `knowledge-base/novels/流浪地球/pipeline/phase1_world.yaml`。

**注**：流浪地球是"科幻 + 末世"杂交。Phase 1 选定主导 driver 时，按故事重心选——若重心在物理学约束 / 技术决策 → `scifi`，若重心在末世生存 → `apocalypse`，但不允许同时挂两个。

## 平台样态参考（platform_mode）

观察时间：2026-05 范围。

- **起点 / 番茄常见钩子**：丧尸 / 尸变 / 异能进化 / 高温末世 / 菌菇世界 / 重生囤物资 / 末日工厂 / 末世种田 / 系统流末世
- **知乎盐言常见钩子**：高温末世（冰元素婚配） / 末世签到 / 末世异能（情感强 hook）

平台样态的共同点：driver 前置到标题或简介，第一段就让读者掌握"威胁是什么 + 主角的反制资源是什么"。

## 高频禁区

| 禁区 | 表现 | 修正 |
|------|------|------|
| 用结果状态冒充 driver | "基础设施崩溃型末世" / "低信息断供" / "秩序瓦解" / "物资匮乏"（SKILL.md §3 末世 4 反例） | 选具体灾变成因 |
| 未声明就突然新增第二种威胁 | 前文是孢子感染，中段冒出军方特工 / 异变生物 / 神秘组织 | driver 选定后再加新威胁须升级到主 driver 或归在 world_rules 派生 |
| 救援 / 补给"凭空出现"无代价 | 主角突然发现完整罐头库 / 救援队空降 / 政府广播 | 任何资源在 world_rules.physical 或 driver 派生里说明来源 + 获取代价 |
| 把"原因隐去"当默认 | 没特别选 classic_mode 就让灾变成因模糊 | classic_mode 是显式选择，需 Phase 0 / Phase 1 明确切换 |
