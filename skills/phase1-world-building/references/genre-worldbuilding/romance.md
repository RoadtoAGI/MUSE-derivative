# 言情（romance）世界观参考

> 本文不复述 SKILL.md §3 的通用规则。言情世界核相对轻，本文重点是"关系阻力的结构性来源"——这是言情 driver 的真正抓手。

## generative_driver 选型范畴

言情 driver 的核心是 **关系阻力 + 关系运作机制**：

- **阻力的结构性来源**：什么让两人不能简单在一起——家族 / 阶级 / 身份 / 契约 / 距离 / 心理障碍 / 时代规范 / 死亡牵引
- **关系运作机制 / 社会脚本**：本世界里"恋爱 / 婚姻 / 性 / 友谊"如何运行；约会的可行场景；社会评价场如何施压
- **误读 / 误会的触发条件**：什么让相爱的人误解彼此；信息差从哪来
- **空间作为阻力载体**：物理隔离 / 阶层壁垒 / 制度边界如何让关系变难

言情**不需要**重世界核（不像末世 / 科幻 / 仙侠那样的硬规则系统），但需要**关系阻力的具体形态**——抽象写"两人有阻力"是空的，必须具体到"什么阻力 / 在哪发生 / 如何运作"。

### platform_mode vs classic_mode

| 模式 | 特征 |
|------|------|
| **platform_mode** | 阻力前置到题眼（"霸总 vs 灰姑娘" / "豪门继承" / "契约婚姻"）；高糖反差 | 起点女生 / 番茄言情 / 盐言短篇 |
| **classic_mode** | 阻力是社会脚本 / 心理障碍 / 死亡牵引等内化的不可逾越物；含悲剧可能 | 严肃文学言情（毛姆 / 村上 / Williams） |

## 名著锚点（few-shot）

### 《挪威的森林》— 死亡牵引型 / classic_mode

driver 拆解（关系阻力 = 死亡 + 精神疾病的牵引）：

- **阻力**：直子受死亡牵引（爱木月但身体不回应 → 身体回应渡边但无法爱），渡边的"死作为生的一部分永存"认知使他无法投入任何事
- **运作机制**：阿美寮（与世隔绝的疗养院）是直子的栖身处；东京是渡边的现实生活场——两个空间的物理隔离 = 精神隔离
- **误读触发**：直子的"想说什么但找不到词"是病理也是存在论失语；渡边以为给陪伴就够，但直子需要的是被吸进她的世界
- **空间作为阻力载体**：东京的城市迷宫（散步成为情感交流） / 阿美寮的电车+巴士隔离 = 关系阻力的实体化

派生 world_rules 关键条：
- 物理：东京无限延展（散步即交流）；阿美寮山中隔离；季节变化与情感节拍同步
- 社会：60 年代末学运是噪音背景；学生宿舍是微型虚假社会；阿美寮的"正常即承认不正常"
- 心理：死作为生的一部分永存；精神疾病有遗传 / 传染性；语言障碍 = 存在危机

参考产物：MUSE-canon-distill `knowledge-base/novels/挪威的森林/pipeline/phase1_world.yaml`。

### 《月亮与六便士》— 创作冲动 vs 体面社会 / classic_mode

driver 拆解（关系阻力 = 艺术家的绝对自我 vs 社会规范）：

- **阻力**：史特利克兰被不可抗拒的创作冲动控制——抛弃妻子 / 朋友 / 道德义务；他不反叛社会，根本不在乎社会
- **运作机制**：维多利亚体面社会 = 中产阶级对名声 / 责任 / 良知的绝对要求；"良知是社会植入个人心中的暗探"
- **空间分层**：伦敦（体面中产）→ 巴黎蒙马特（波希米亚贫困）→ 马赛（底层流浪）→ 塔希提（原始自然）——空间下沉对应社会脚本的逐层剥落
- **关系毁灭机制**：施特洛夫的善良无力保护自己被践踏；勃朗什被原始吸引力摧毁；艾塔以原始忠诚填补

参考产物：MUSE-canon-distill `knowledge-base/novels/月亮与六便士/pipeline/phase1_world.yaml`。

### 《斯通纳》— 制度 + 心理被动型 / classic_mode

driver 拆解（关系阻力 = 学院制度 + 性别阶级壁垒 + 心理被动）：

- **阻力**：斯通纳天性中的被动与忠诚的矛盾——"你太弱了，你又太固执了"
- **运作机制**：学术等级（助教→助理教授→正教授）；系主任以课程分配为武器；伊迪丝的冷暴力控制；20 世纪初中西部清教伦理（婚外情 = 学术生涯终结）
- **关系毁灭**：凯瑟琳作为年轻女性助教极易成为"道德"借口的牺牲品 → 斯通纳放弃凯瑟琳保护她的职业
- **沉默即同意**：农民家庭心理模式 → 关键抉择全是被动接受

参考产物：MUSE-canon-distill `knowledge-base/novels/斯通纳/pipeline/phase1_world.yaml`。

## 言情的"轻世界核 + 强关系结构"特征

言情不像末世 / 科幻 / 仙侠那样需要重世界规则。Phase 1 言情的输出重心在：

- **关系阻力的具体形态**（>>> 物理 / 社会规则的细密度）
- **空间作为阻力载体**（关系阻力常通过空间隔离体现）
- **社会脚本的运作**（什么样的恋爱被允许 / 什么被禁止 / 什么会被审判）

写作时 Phase 5/6 主要消费的是 driver 中的"关系阻力 + 社会脚本"——`world_rules.physical` 通常较少，但 `world_rules.social` 和 `world_rules.psychological` 应丰富。

## 平台样态参考

观察时间：2026-05 范围。

- **起点女生 / 番茄常见钩子**：现代言情 / 古代言情 / 豪门虐恋 / 契约婚姻 / 双向暗恋 / 系统流言情 / 重生甜宠 / 强势女主
- **盐言常见钩子**：短篇高反转言情 / 一夜情后双向救赎 / 错过 / 救赎向 / 暗恋公开
- **平台样态特征**：阻力前置（"豪门" / "霸总" / "继承人" / "契约"）；反转密度高（短篇尤甚）；社会评价场被淡化（爽文向）

## 高频禁区

| 禁区 | 表现 | 修正 |
|------|------|------|
| 阻力悬空 | 写"两人有阻力"但不说阻力具体是什么 | driver 必须具体到"什么阻力 + 在哪发生 + 如何运作" |
| 把"误会"当 driver | 整本书靠"如果他早一点说就好了"推进 | 误会是机制，不是 driver；driver 是产生误会的结构性条件（信息差 / 身份隐瞒 / 阶级壁垒）|
| 社会脚本被忽略 | 现代中产或古代宫廷恋情，但不交代社会评价场如何施压 | `world_rules.social` 写清：什么样的关系会被审判 / 什么样的会被允许 |
| 阻力靠"突然出现的反派" | 关系阻力靠"第三者 / 反派 / 突发事件" | 阻力应是结构性的（家族 / 阶级 / 身份 / 制度 / 心理）——单一反派只能作机制层 |
| 重世界核冗余 | 写大量物理规则 / 武学体系（言情套科幻 / 仙侠壳但不做杂交） | 言情主导时 driver 重心在关系阻力 + 社会脚本；其他规则按"够用就好"原则 |
