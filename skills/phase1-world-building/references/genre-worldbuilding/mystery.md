# 悬疑（mystery）世界观参考

> 本文不复述 SKILL.md §3 的通用规则。本文承接对照表，给悬疑题材专属的 driver 选型、信息边界与高频禁区。

## generative_driver 选型范畴

悬疑 driver 必须回答四件事：

- **隐藏事件**：被埋藏的真相是什么——作案细节 / 历史真相 / 失踪事件 / 系统性犯罪
- **作案手法 / 事件运作机制**：怎么发生的——物理可行性、谁有手段、需要什么前提
- **线索泄漏路径 / 信息拓扑**：哪些线索在何时何地以何种方式可被发现；谁知道什么，谁不该知道什么
- **公平性边界**：读者获得的信息是否足以推出真相（fair play）；reveal 不可凭空给

driver 选定后，Phase 5/6 才知道每场戏要泄漏哪些线索 / 隐瞒哪些信息 / 给读者哪些误导。

### platform_mode vs classic_mode

| 模式 | 特征 |
|------|------|
| **platform_mode** | 案件类型前置（密室 / 连环 / 倒叙 / 推理流）；侦探 / 线索 / 嫌疑人格式清晰 |
| **classic_mode** | 颠覆侦探叙事，把"追查"本身变成对世界本质的隐喻；可不解决 |

主流悬疑用 platform_mode；严肃文学型 / 反类型作品用 classic_mode（如下面的 2666）。

## 名著锚点（few-shot）

### 《2666》— 系统性暴力 / 反类型 / classic_mode

driver 拆解：

- **隐藏事件**：圣特莱莎（影射华雷斯城）的妇女连环谋杀案；阿琴波尔迪（神秘德国作家）的真实身份
- **作案手法 / 事件运作机制**：罪行运作在权贵 / 毒贩 / 警察的利益链条中；体制性掩盖；外来者（评论家、记者）能看见但无法改变
- **线索泄漏**：第四部用编年体罗列受害者档案——线索是累积的，不是收束的
- **公平性边界**：故意颠覆 fair play——**没有侦探，没有破案**；阿琴波尔迪始终未被找到；罪行未被解决

派生 world_rules 节选：

- 物理：边境城市 / 沙漠 / 工厂 / 加工出口区
- 社会：权力链条（政客 + 毒贩 + 警察） / 底层女性是主要受害者 / 知识分子有特权无行动力
- 心理：面对恶逃避是默认 / 艺术能记录但无法阻止

参考产物：MUSE-canon-distill `knowledge-base/novels/2666/pipeline/phase1_world.yaml`。

**典型反类型选择**：故意不给侦探角色、故意不给 reveal、故意让汇聚不产生解决。classic_mode 的悬疑作品须像 2666 一样，**显式选择**颠覆点（Phase 0 `originality_statement.cliches_to_avoid` 中应明示）。

## 平台样态参考（platform_mode）

观察时间：2026-05 范围。

- **起点 / 番茄常见钩子**：刑警系列 / 法医系列 / 密室连环 / 倒叙手法（先看到结局，倒推过程） / 推理流（系统视角） / 灵异悬疑（带有超自然元素，但本质是悬疑推理）

## 高频禁区

| 禁区 | 表现 | 修正 |
|------|------|------|
| 用案件"类型"当 driver | "密室悬疑" / "连环悬疑"（类型分类，不是 driver） | driver 应到事件 + 运作机制 + 线索路径具体层 |
| 线索越权曝光 | 嫌疑人内心独白直接 leak 真相 / 旁白告诉读者凶手是谁但侦探不知 | 信息分发须遵守 fair play 边界——读者拿到的 ≤ 侦探拿到的 + 公平推理空间 |
| reveal 凭空给 | 没有前置线索就直接揭示真相 | reveal 的每条都应能 trace 到前文已露的线索（除非显式 classic_mode 反类型） |
| 误导靠隐瞒不是误读 | 故意不写关键事实让读者猜不到 | 误导应靠**已写出的线索可被合理误读**，不靠**隐瞒线索** |
| 把"想得多做得少"当 driver | 描述知识分子无力感作为悬疑核心 | 那是 world_rules.psychological 层；driver 仍是具体事件 + 运作机制 |
