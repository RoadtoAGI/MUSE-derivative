# Prose-Craft — 散文写作工坊

## 核心原则

> 「节拍是人物行为中动作/反应的一种交流。」

> 「任何文本都有其潜文本。」
> —— 《故事》第十一章

节拍是叙事的最小单位——一个行动和一个反应的交换。场景由节拍链构成，节拍链驱动价值从开始状态转变到结束状态。对白不是交谈，而是行动——每句话背后都有一个意图，字面意思下藏着潜台词。

**信息让位于行动**：新机制 / 物证 / 规则 / 判断可以多，但必须分批进入并绑定行动——逻辑链不能淹没行动链和情绪链。读者不是在读侦查报告，而是在跟随人物在压力中行动。

## 写作核心（L1 — 写作前必须理解）

以下三条是散文写作最核心的原则。其余所有指导都服务于它们。

**① 节拍驱动**：

> 「第三步：将场景分解为节拍。从两个层面仔细审查场景的第一个动作：从外在而言，看人物表面上是在做什么，而且更为重要的是，还要透过这一表面，看到他实际上是在做什么。」
> —— 《故事》第十一章

场景由行动/反应的节拍链构成，节拍链驱动价值从 `value_start` 转变到 `value_end`。同一个行为模式重复属于同一个节拍——直到行为发生明显变化，才进入新节拍。节拍无需事先单独设计，在写作中自然展开即可。

**② 潜文本三层**：

> 「已说是角色选择向别人表达的想法和情感；未说是角色这些想法和感觉用内在声音只表达给自己的部分；不能说的，是角色的潜意识冲动与欲望不能说出来者。」
> —— 《对白》前言

每句台词都是一个**言语行动**（说服、威胁、安慰、试探...），字面意思和真实意图之间应有间距。人物不会直接说出内心想法。

**③ 省略决策**：

不是每个设计动作都需要出现在正文里。如果一连串动作只传达同一个信息，只保留最有代价感的一个，其余省略。读者能自行补完的物理过程不需要逐一展示。

> 「同一节拍重复又重复，戏就变得平板沉闷。」——《对白》第九章

## 写作指导（L2 — 写作中遵循）

**对白**（writer / 单场景任务通用）：

- 参照 Phase 2 的声音特征确保角色差异化
- role_brief 的 `desire_now` / `fear_now` / `misread_now` 应在对白的潜台词层有投射——此刻的欲望、恐惧、对他者的误读，会影响角色选择什么行动、说什么话、避开什么话
- 深入对白设计时加载 `dialogue-craft` skill（对白矩阵 / 声音差异化 / 质量验证 / POV 锚点）

**叙述**：

- **场景描写**：选一两个细节深入，不做资产盘点
- **动作描写**：能用行动表达的，不要用内心独白
- **心理刻画**：深度不能超过角色本人的认知水平

**正文隔离**：设计文档和创作辅助材料不得泄漏进正文。Phase 4 结构标注、Phase 5 场景设计、role_brief、麦基理论引用——读者绝不应在正文中看到它们的痕迹。设计文档告诉你"这场戏要从信任走到背叛"，你要通过行动、对白和细节让读者**自己感受到**这个变化。

**段落节奏**：段落和节拍不是一一映射。一个段落可以承载多个连续的 beats——尤其是长景深段落（主要冲突展开、关键动作全过程、长对话推进）承载 3-5 个 beats 是正常的。真正该换段的是焦点、视角、时间、或价值的转折。单句成段留给鸿沟裂开的瞬间。长段的具体长度由题材和场景压力决定，不套固定句数。

**迟进早出**：尽可能晚地进入场景，目的达成后立即结束。

**克制**：同类修辞手法建立模式后效果递减。情感强度必须匹配情境压力。重大情感事件（重逢、所爱遇险、信任翻转、秘密揭露）反应过淡 = 人物像冷漠机器人；过烈 = 矫情；都属于反应失真，常因任务密度过高、外部动作把 POV 反应挤掉。

- ❌ 「她抬眼看见他，未喊，抛出湿绳。」（外部动作完成事件，POV 反应被挤掉）
- ✅ 「她抬眼，灯下侧脸像多年前的剪影，旧弦轻一颤，她压住将出口的呼，把船舷湿绳挑起。」（身体反应 + 记忆触发 + 压制 + 落到动作）

**双层风格控制**：
- **作品风格**（Phase 0 `style_directives`）：全局基调
- **角色声音**（Phase 2 `voice_traits`）：局部差异
- 两者冲突时，以作品风格为框架，角色声音在其内部做差异化

**领域知识融入**：如果 Phase 1 提供了 `domain_knowledge`，在叙述中有机嵌入（角色的动作、观察、内心感受），不生硬罗列。

**信息暴露**：背景信息应在冲突中自然流出，而非通过说明段落。

## 名著手艺操作库（每场最多主用 1-2 个）

11 条操作的完整说明见 [`references/novel-craft-patterns.md`](references/novel-craft-patterns.md)。短列表：

1. 程序 / 技术文体替代抒情
2. 身体感觉替代思想阐释
3. 物件承载权力 / 证据 / 关系
4. 受限 POV
5. 日常化宏大议题
6. 省略顿悟内容
7. 期望颠覆链
8. 尺度收缩
9. 他者之眼
10. 句法即症状
11. 创伤后置 / 延迟疼痛

**硬约束**：每个操作必须回答：它替代了哪段本来会出现的解释 / 心理描写 / 背景说明？

**新硬约束**（与 role-brief carrier_hint 联动）：若 role_brief 的 `carrier_hint.role_in_carrier` 标了该角色是 actor / object_holder，则操作选择必须呼应 carrier；不允许选与 carrier 无关的操作。

## 创意点 → 可感材料转化原则

创意点必须先转化为小说中的可感材料：动作、对白、物件、场景压力、叙述距离或省略。不要把创意说明原样交给读者。

每个创意点都要回答三个问题：

1. 它让读者**看见**了什么变化？（不是告诉读者，是让读者看见）
2. 它**替代**了哪段解释？（用动作 / 物件 / 沉默替代说明）
3. 它**为什么只能在这个场景出现**？（场景独占性是创意点价值的检验）

**写后自检**：策划清单、人物小传、结构术语是否被逐字搬入正文，而没有转化成行动、对白、物件或视角限制？是 → 重写该段。

## 写后自检（L3 — 每个场景写完后校对）

写完一个场景后，快速校对以下高频问题。这些不是写作时需要时刻记住的规则，而是写完后的检查清单。

**AI 陈腔核心层**（详见 `references/ai-cliche-patterns.md`）：
- **主次失焦**：节拍主体动作正在进行时，是否转去修饰次要对象？
- **无功能修辞**：删掉某个比喻后，读者对场景的理解是否减少？如果不减少，删掉
- **格言金句**：动作段落间是否插入了对称句式的"主题升华"？
- **情绪库存短语**：是否用了"血液冰凉"、"心脏一缩"等高频套句？
- **描写→说话强制收束**：段落是否总以"说话动作+对话"作为出口？
- **题材盲均质分段**：段落长度是否高度均匀？对照 ref 文件的密度元数据
- 段落是否密集出现短分句 / 逗号过密？
- 连续动作短语是否被合并为场面动作 / 转为人物选择？
- 是否反复使用「像某种 X / 一种 X / 没有 X 的 Y」这类短比喻债务？
- 是否反复使用「没说话 / 停了一秒 / 看了一眼 / 嘴角动了动」等沉默停顿模板？
- 社交场景中是否出现「说了什么 / 跟人说话去了 / 接电话去了」等社交调度流水？

**正文隔离检查**：Phase 5 的 scene_tasks / handoff / role_brief 是否被逐字搬入正文而未经戏剧化转化？

**创意执行检查**：
- `[灵感]` / `[惊艳]` 条目的戏剧意图在正文里有间接呈现吗？还是只复述了条目字面？
- `boldness_guardrails` 所指的强度和正文基调匹配吗？高激活却写平淡，或弱激活却 neurotic？
