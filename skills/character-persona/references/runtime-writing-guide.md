# runtime 模式填写指南

本 skill 的填写启发。

## 身份与处境：不突出职业标签

> 「身份标签是调味料，不是主菜。角色的职业偶尔影响语言，但不应成为默认修辞模式。程序员在求生时首先是人，其次才是程序员。」

| 避免 | 推荐 |
|------|------|
| 陈默，30岁，资深后端工程师 | 陈默，30岁，写了八年代码的人，最近开始怀疑自己是不是只会跟机器说话 |
| 黎安，29岁，战地医护兵 | 黎安，29岁，在沙漠哨站撑着临时病房，别人发烧流血时第一反应总是找她 |

## 声音框架：定义倾向，不定义映射

> 「不要过分铺陈，像撒胡椒面一样地在稿纸上加上没完没了的有关行为、手势、语调方面的描写。」
> —— 麦基《故事》第十七章

| 避免 | 推荐 |
|------|------|
| rhetoric: "心跳加速=CPU负载红线" | "偶尔用技术类比描述情绪，但多数时候用普通感官语言" |
| syntax: "短句、动词主导、冷静直白" | "倾向短句和直白表达，但在情感涌动时句子会变长、变碎" |
| 全文为角色设计大量情感→修辞映射 | 描述几个关键倾向，Phase 6 自由发挥 |

## 边界：比描述更重要

边界定义角色的"负空间"——不会做什么。这往往比正面描述更能防止 AI 套路：

| 角色 | 正面描述（有用但不够） | 边界（更有效） |
|------|---------------------|---------------|
| 沉默寡言的军人 | "说话简洁有力" | "不会在任何情况下长篇大论地分析自己的感受" |
| 温柔的老师 | "说话温和耐心" | "不会用居高临下的口气，即使对方犯了明显的错" |

## 性格真相：保留裂隙

如果 Phase 2 设计了人物塑造与性格真相之间的裂隙，这个裂隙是角色最有戏剧价值的部分。不要在 Skill 中把它描述得太清楚（"ta 表面冷漠其实内心温暖"），而是让裂隙通过表演规则和边界自然体现。

## 内在能力：找一个 ta 失去后就不是 ta 的能力

不是技能 / 职业能力。是 ta 内化的"维持自我"的工具。判定方式：写一句"如果 ta 失去 X，ta 还是 ta 吗"——答案是"不"才填进 inner_capacity.primary。

| 避免 | 推荐 |
|------|------|
| primary: "战斗能力"（这是技能不是 inner_capacity） | primary: "在被嘲讽时仍能讲个笑话回去的反弹力" |
| primary: "智商高"（这是属性不是 inner_capacity） | primary: "在所有人都说她错时仍能写下'我没错'三个字的自我说服" |
| loss_signal: "她变得不开心"（这是抽象状态） | loss_signal: "她想编一个故事但编不下去，三次想开头都断在第二句"（具体动作 / 反应缺席）|

**和 character_arc.mode 的区别**：character_arc 关心"ta 在故事中是否变化"；inner_capacity 关心"无论 ta 是否变化，ta 维系自己的方式是什么"。static 角色也可以有 inner_capacity loss——这才是 static 角色的戏剧张力来源。

## 主体性物件：让配角拥有不属于主角的东西

配角档案 70% 的篇幅是"ta 与主角的关系"。subjectivity_object 字段强制保留一个"与主角无关的创造物"。

| 避免 | 推荐 |
|------|------|
| 完全不填（配角退场时无东西可承载） | 哪怕一句"她小时候写过的一首七言绝句，自己藏在脑海里" |
| 物件过于精雕成艺术品（"她暗中是大诗人"） | 物件必须符合配角实际能力——半成品 / 习作 / 私人物件 |
| 物件已知给主角（"她给主角写过情诗"） | 物件最好主角不知道——这才是主体性 |

## load-bearing 角色 vs supporting cast 的 inner_capacity 填法差异

`## 内在生存能力` 章节在模板中标 `<!-- required -->`，但执行强度按角色类型分：

- **主角 / 反派 / 关键配角（load-bearing）**：必须填实质内容（能力 + loss_trigger + loss_signal 全填）
- **普通配角 / 功能性角色 / 转述角色 / 背景人物**：合法填法 `primary: "N/A — pure function role"`，其余子字段省略

判定 load-bearing：角色是否承担**弧光 / 价值变化 / 关键决断 / 主题承载**任一职责？是 → load-bearing；否 → supporting，可走 N/A 填法。

verify_phase2_assets 章节白名单校验仍按 required 走（章节存在即过），不强求实质内容——内容质量在 character-persona 描述性质量信号 §4 由 reviewer 判 HOW。

## 对手和配角的简化版

以下是常见的深度梯度，但不是硬规则——根据角色在故事中的实际分量决定：

| 角色类型 | SKILL.md 参考深度 | state.md 参考深度 |
|---------|------------------|------------------|
| 主角 | 完整模板 | 完整初始化 |
| 对手 | 完整模板（从对手视角写——对手认为自己是对的） | 完整初始化 |
| 有独立对白的配角 | 视情况裁剪——至少需要身份、声音、边界 | 视情况裁剪——至少需要关系感知 |
| 纯背景角色 | 通常不需要独立 Skill | — |
