# 叙事一致性审查（B 组）

> 审查目标：检测全文叙事层面的自相矛盾——角色行为是否前后一致、事实细节是否稳定、叙事风格是否统一。
>
> 这些问题往往分散在相距较远的段落中，在逐场景写作时不易察觉，但读者通读全文时会立即注意到。

## 输入

**主场景（Phase 6 → Phase 7 过渡）：**
- `pipeline/scenes/scene_{id}.md` — 全部场景正文，按顺序通读（核心依赖）
- `pipeline/characters/{角色名}.md` — 角色设定档（参考依赖，用于对照角色设计基准）

**副场景（Phase 7 之后二次审查，story.md 已存在）：**
- `story.md` — 替代场景文件作为正文来源（核心依赖）
- `pipeline/characters/{角色名}.md` — 同上

## 审查维度

### 1. 角色一致性（characterization）

检查角色的记忆、知识、能力、身份在全文中是否自相矛盾。以 `characters/*.md` 中的设定为基准。

**扫描要点：**

**A. 记忆与身份连续性**
- 角色遗忘了已建立的关系、经历或承诺
- 角色回忆起文本中从未发生的事件
- 个人历史、家庭背景的前后不一致
- 角色对同一段往事的回忆在不同场景中细节不同（地点、人物、时长）
- 角色"知道"或"不知道"某件事的状态前后翻转，且无信息获取情节支撑

**B. 知识矛盾**
- 角色展现出超越其背景设定的知识或词汇
- 角色本应知道的信息却表现得不知道
- 时代/文化背景错位（古代角色用现代概念等）

**C. 能力与具身约束**
- 角色能力在不同场景间剧烈变化且无叙事解释
- 已建立的技能在需要时莫名消失
- 角色做出远超或远低于其设定能力的行为
- 身体状态与行为冲突——前文建立的伤势、疲劳、物理限制在后文被无声忽略（如手臂受伤的角色下一场景正常使用双手）

**D. 内在能力连续性（inner_capacity_continuity）**

人物档案中标注的"内在生存能力"（幻想 / 写作 / 友善 / 信仰 — 见 `character.md` 的 `inner_capacity` 字段）在创伤场景后是否连续：

- 角色被设计为"幻想能力强"但创伤后立刻恢复幻想 = 失守
- 创伤场景描述了能力失灵但后续场景能力悄然回归 = 失守
- 排除项：长期康复后的逐步恢复（必须显示恢复过程）

参考：《你好，旧时光》场景 05 → 余周周的幻想能力失灵后续多场景仍在影响（不是单场景一次性创伤）。

**E. 延迟揭示型死亡的渗透连续性（delayed_death_seepage）**

若 Phase 5 `scene_card` 设计了某角色"延迟揭示型死亡"（不写死亡现场，让死讯通过事后碎片渗透），B 组核对：

- 死亡现场是否真的没写（写了 = 失守）
- 事后碎片是否累积到揭示阈值（碎片太少 → 读者不会"突然意识到"）
- 死讯被一次性大段揭示（角色突然大段回忆死讯）= 失守，应分多场景渗透

参考：《你好，旧时光》场景 14 → 通过公交车瞌睡、烧纸钱、午夜惊醒等多场景碎片让读者意识到妈妈已去世。

**F. 不可靠叙述者一致性（unreliable_narrator_consistency）**

若 Phase 2 / Phase 3 设计了第一人称叙述者具有不可靠性（结合 `scene_card.narrator_distance` 在场景层的体现，如《流浪地球》末世叙述者已倒向叛军未自觉），B 组核对：

- 叙述者是否在道德灰区场景中保持"未自觉"（出现"但我隐约觉得不对" / "事后回想我感到羞耻" = 失守）
- 叙述者的不可靠是否在某些场景突然变成可靠（无设计支撑）= 失守
- 设计意图与读者最终发现的反差是否得到保护

参考：《流浪地球》场景 14 → 叙述者目睹处刑感到"快感"，全文未加任何自觉标记。

**排除项（不是真正的矛盾）：**
- 角色成长、学习、疲劳导致的合理能力变化
- 情绪状态影响表现（恐惧、愤怒、决心）
- 不可靠叙述者的有意设计
- 比喻性或诗意的表达
- 角色出于道德/策略原因选择不使用某能力

### 2. 事实细节（factual_detail）

检查具体事实在全文中是否前后一致。这类错误最容易被读者发现，也最容易修复。

**扫描要点：**

**A. 外貌描述**
- 发色、眼色、身高、体型等物理特征前后不一致
- 服装、饰品在同一时间线内突然变化（无换装情节）
- 伤疤、标记等特征时有时无

**B. 命名与语义锚点**
- 角色名、地名、物品名的拼写或称呼不一致
- 同一对象在不同位置被赋予不同名称（非有意的别名）
- 称谓关系错误（如称呼父亲为叔叔）
- 专有名词、规则条文、战术名称、脚本措辞等"稳定锚点"在全文中悄然漂移——前文建立的精确表述在后文被改写成不同版本

**C. 数量与度量**
- 数量前后矛盾（"三个兄弟"后文变成"两个兄弟"）
- 距离/时间度量不一致
- 物品数量在场景间凭空增减
- 留意全文的"数量账本"：人数、剂量、次数、文件份数、库存等在多处提到的数字，检查它们加起来是否自洽

**排除项：**
- 角色主观视角导致的描述偏差（如角色记错了某人的发色，是有意设计）
- 比喻性描述（"她的眼睛像火焰"不是说眼睛是红色的）
- 合理的环境变化（光线变化导致颜色感知不同）

### 3. 叙事风格（narrative_style）

检查叙事视角和文体风格在全文中是否保持一致。风格漂移会破坏读者的沉浸感。

**扫描要点：**

**A. 视角混乱**（**跨场景**规则——Step 5 分工：A 组 §8 专注**单场景内**焦点角色感知边界；B §3.A 专注跨场景视角切换规则）
- **跨场景无动机切换 POV**——场景 A 是 X 的限制视角，场景 B 突然切到 Y 却无任何过渡 / 分隔标记
- 第一/第三人称混用（非有意的叙事手法）——通常跨场景出现才是漂移
- 全知视角和限制视角之间无标记切换——**跨场景**的叙述层级漂移
- **单一场景内**的"POV 角色不可能知道的信息" → 归 A 组 §8，不在本节重复

**B. 语调突变**
- 严肃叙事中突然插入不协调的幽默（或反之）
- 叙述者的情感距离突然改变（亲密 → 疏离，无过渡）
- 文学性与口语化程度在相邻段落间剧烈波动

**C. 风格漂移**
- 句式结构的复杂度突然改变
- 描写密度（感官细节的丰富程度）前后不一致
- 叙述节奏在类似场景中差异过大

**排除项：**
- 不同 POV 角色之间的风格差异（这是好的设计）
- 有意的风格对比（如回忆与现实的不同叙事质感）
- 故事内部时间推进带来的自然语调变化
- 高张力场景中句式变短（这是正常的节奏控制）

## 输出

按 [`output-schema.md`](output-schema.md) 格式返回。一致性维度的发现应尽量提供 `contradiction_pair`——引用矛盾的两端，让 orchestrator 能快速定位并判断以哪一端为准。

`source` 字段统一为 `"story"`（本组不检查 pipeline 文件）。
